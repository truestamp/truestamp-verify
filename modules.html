    Offline version of verify().

    In offline mode, if no keys are provided, the library will attempt to use a backup copy of the keys stored in this library. These backup keys are not guaranteed to be current, but they are the best available option.

    In offline mode, the library will not attempt to verify transactions against the actual on-chain state. It will only verify that the commitment is internally cryptographically sound. Since it does not have access to the on-chain state, it cannot verify or display a timestamp attested to by any transactions in this commitment. You can still use the transaction information provided to manually (with your eyes) verify the transaction against the on-chain state using, for example, a block explorer.
